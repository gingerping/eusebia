System Overview

What is the EPAMNHS Portal?

A PHP-based school management system with three distinct user roles

System Name: Eusebia Paz Arroyo Memorial National High School Portal (EPAMNHS)
Built with PHP (OOP), MySQL, Bootstrap 5, and Chart.js. Manages student enrollment data across Grades 7–12, with separate interfaces for Administrators, Staff, and Students/Residents.

login()
Accepts email/phone + password. Checks against three user tables: admin, staff, and resident. On success, stores user data in $_SESSION and redirects by role.
classes/main.class.php : login()
🛡️
validate_admin()
Called at the top of every admin page. If the session does not contain admin credentials, it redirects the visitor to login.php immediately.
classes/main.class.php : validate_admin()
🚪
logout()
Destroys the active session and redirects to login.php. Triggered from the sidebar dropdown in both admin and resident layouts.
logout.php → main.class.php : logout()
🗂️
get_userdata()
Reads the current session and returns an associative array with the logged-in user's ID, name, email, and role — used on every protected page.
classes/main.class.php : get_userdata()
// Simplified login flow (main.class.php) public function login() { if (isset($_POST['login'])) { // 1. Check tbl_admin → set session role = 'admin' // 2. Check tbl_staff → set session role = 'staff' // 3. Check tbl_resident → set session role = 'resident' // 4. Redirect to appropriate dashboard } }

Role Routing: After successful login, the system reads the matched table and routes: Admins → admn_dashboard.php, Staff → admin area, Students → resident_homepage.php.

Same password-change flow for admin accounts. Validates current credentials before accepting the new password, preventing unauthorized resets.

Fetches all fields for a specific resident by id_resident. Used to pre-populate the profile edit form with existing data before the user makes changes.

Architecture Pattern: The system follows a simple MVC-like structure. PHP class files in classes/ act as the model layer. Each .php page is both controller and view. The shared sidebar is split into dashboard_sidebar_start.php and dashboard_sidebar_end.php and included on every admin page.
